Extensions to the Java(TM) Remote Method Invocation (Java RMI) activation framework to support activatable objects that are exported using an {@link net.jini.export.Exporter} to go inactive, to provide invocation handlers for exporters that want to use dynamic proxies for activatable remote objects, and to support trust verification of secure activatable dynamic proxies and secure activation identifiers.

Some of these extensions can only be used in conjunction with an activation system daemon or activation group that supports them. The standard rmid and its default activation group do not support them. @since 2.0 @version 2.0
